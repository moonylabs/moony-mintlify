<div style={{ textAlign: 'justify' }}>

<p style={{ marginBottom: '1.5rem' }}>
Moony's price discovery is governed by an onchain bonding curve, a deterministic mathematical function encoded within the Moony Reserve. The pricing curve adjusts the token's spot price based solely on the current absolute supply of MNY in circulation.
</p>

<p style={{ marginBottom: '1.5rem' }}>
This mechanism applies symmetrically to both unlocking and redemption. When MNY is unlocked from the pre-minted supply, the pricing curve determines the required USDF contribution according to the current circulating supply. When MNY is redeemed, the same curve determines the USDF returned. All participants interact with the same pricing function, enforced uniformly by the contract's immutable logic.
</p>

<p style={{ marginBottom: '1.5rem' }}>
As circulating supply increases, the bonding curve raises the price in a non-linear fashion. This structure rewards early participation through lower entry costs while requiring later participants to contribute proportionally more liquidity. These price differences arise mechanically from the public state of the system at the time of interaction, not from identity, special access, or discretionary allocation.
</p>

<p style={{ marginBottom: '1.5rem' }}>
Because the bonding curve operates transparently and permissionlessly, access to MNY is governed by open, predictable rules rather than gatekeeping or administrative control. The pricing curve ensures that growth reflects real demand, reinforces scarcity over time, and aligns economic participation with the long-term integrity of the network.
</p>

</div>

## Model Parameters

<div style={{ textAlign: 'justify' }}>

<p style={{ marginBottom: '1.5rem' }}>
The bonding curve implemented in the Moony Reserve uses a discrete bonding curve implementation, where pricing values are sampled from an underlying exponential function at fixed supply intervals. This approach enables efficient on-chain computation through precomputed lookup tables.
</p>

<p style={{ marginBottom: '1.5rem' }}>
The underlying mathematical model defines the spot price (the instantaneous price per MNY token) at any given level of circulating supply using an exponential function. Rather than computing this exponential function and its integrals on-chain for each transaction, the protocol uses precomputed lookup tables that contain discrete values sampled from this mathematical curve at fixed supply intervals. These tables enable fast, gas-efficient pricing calculations while maintaining the exact same curve shape and pricing behavior defined by the mathematical formulas below.
</p>

<p style={{ marginBottom: '1.5rem' }}>
When a participant unlocks or redeems MNY, the Moony Reserve determines pricing by looking up values from these precomputed tables, which are derived from integrating the spot price function over the relevant supply intervals. This discrete implementation ensures that pricing reflects the exact position and size of each transaction along the pricing curve while minimizing on-chain computation costs.
</p>

</div>

## Cost Calculation

<div style={{ textAlign: 'justify' }}>

<p style={{ marginBottom: '1.5rem' }}>
The mathematical formulas below define the continuous bonding curve from which the discrete lookup tables are generated. These formulas can be used to verify the precomputed values in the lookup tables or to compute prices analytically.
</p>

<p style={{ marginBottom: '1.5rem' }}>
When unlocking or redeeming MNY through the Moony Reserve, pricing is not applied at a single fixed point. Instead, each transaction spans a range of circulating supply levels. The discrete implementation uses precomputed tables derived from integrating the spot price function over supply intervals. The formulas below show how these integrated values are calculated mathematically.
</p>

</div>

<div className="principle-card formula-card" style={{
  padding: '1.5rem',
  marginBottom: '1.5rem'
}}>
  <div className="formula-box">
    $$
    Cost = (a \times b / c) \times (e^{c \times S_{new}} - e^{c \times S_{current}})
    $$
  </div>
  <div className="formula-explanation">
    <p style={{ marginBottom: '0.75rem', marginTop: 0 }}>
      To buy tokens, you pay the integral of the spot price from your starting supply to your ending supply. This gives you the total USDF cost for the tokens you're purchasing:
    </p>
    <div style={{ marginTop: '0.75rem' }}>
      <strong>Where:</strong>
      <ul style={{ marginTop: '0.5rem', marginBottom: 0, paddingLeft: '1.5rem' }}>
        <li style={{ marginBottom: '0.5rem' }}>
          <code style={{ backgroundColor: 'transparent', padding: 0 }}>S_current</code> is the supply before your purchase
        </li>
        <li style={{ marginBottom: '0.5rem' }}>
          <code style={{ backgroundColor: 'transparent', padding: 0 }}>S_new</code> is <code style={{ backgroundColor: 'transparent', padding: 0 }}>S_current + tokens_to_buy</code> (the supply after your purchase)
        </li>
        <li style={{ marginBottom: '0.5rem' }}>
          The formula integrates the spot price function over this range
        </li>
      </ul>
    </div>
  </div>
</div>

<div className="principle-card formula-card" style={{

}}>

    $$
  </div>

    <p style={{ marginBottom: '0.75rem', marginTop: 0 }}>
